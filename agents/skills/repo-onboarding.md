# repo-onboarding

## Purpose

unfamiliar repo やサブディレクトリに入ったとき、最短で安全に入口、コマンド、正本を把握します。

## Use When

- repo の全体像がまだ曖昧
- どの文書を読むべきか迷う
- task 前に前提確認が必要

## Read Order

1. `README.md`
1. `QUICK_START.md`
1. `documents/README.md`
1. `documents/WORKFLOW_GUIDE.md`
1. `docker/README.md`
1. `agents/README.md`
1. Codex なら `agents/canonical/CODEX_WORKFLOW.md`
1. `scripts/README.md`

## Outputs

- repo shape の短い要約
- 触るべきディレクトリ
- 追加で読むべき正本
