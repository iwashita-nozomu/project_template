# 演算子記法の厳格ルール

この章は、演算子記法の厳格な使い分けを定めます。

## 要約

- 線形は `@`、合成は `*`。
- 混在は括弧で明示します。
- Python 実装では禁止事項もここで定めます。

## 規約

- 共通の基本方針は `documents/conventions/common/04_operators.md` を正本とします。
- **線形作用素の適用は `@` を使う**（例: `Mv @ v`）。
- **線形作用素の合成は `*` を使う**（例: `Mv * proj`）。
- 非線形作用素は `Op` を使い、**適用は `__call__`**（例: `op(x)`）。
- `@` と `*` を混在させる場合は**括弧で明示**します（例: `A @ (B @ v)`）。
- `LinOp` / `Op` の内部関数を直接呼び出さず、**演算子記法を優先**します。

## 禁止事項

- **線形作用素を `__call__` で適用しない**（例: `Mv(v)` は禁止）。
- **非線形作用素を `@` で適用しない**（例: `op @ v` は禁止）。
- **合成と適用を曖昧に書かない**（例: `A @ B @ v` は括弧なしで書かない）。
- **内部関数を直接呼び出さない**（例: `LinOp.mv(x)` / `Op.func(x)` は禁止）。
  - 理由: 表記が揺れると、線形/非線形や合成/適用の区別が曖昧になり、レビューと保守が破綻します。
- **型の意味が曖昧な `Array` を引数・戻り値に使わない**。
  - ただし **`python/jax_util/base/` 内部** と **`python/jax_util/base/protocols.py`** は例外とします。

## 投影・前処理の記法

- 投影は `proj @ v` の形で統一し、`proj` は `LinOp` とします。
- 前処理は `precond @ r` の形で統一し、合成は `proj * precond * proj` を基本とします。
- 恒等写像は **既定引数として使用できます**。実装は `LinOp(lambda x: x)` を使います。
- 既定引数を採用する場合は、**投影を省略しても意味が変わらない**ことをコメントで明示します。
